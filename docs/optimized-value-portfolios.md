# Optimized Value Portfolios

*Diversity-maximizing portfolios of synthetic value personas, built on Schwartz's Theory of Basic Values (PVQ-RR).*

> Generated for the `values-explorer` project. Each persona is a synthetic 19-value profile chosen so the portfolio spreads as widely as possible across **both** the value space **and** the stressor-sensitivity space. Every persona below includes a copy-paste **agent system prompt** that encodes its full PVQ-RR scores.

---

## How these portfolios were built

**Goal.** Maximize diversity along two coupled dimensions at once:
1. **Value schemes** — spread across the 19 Schwartz values (scored 0–7).
2. **Stressors identified** — spread across the 12 "stressor" decision-contexts that each value scheme is sensitive to (the constraints that satisfy or frustrate it).

**Stressor sensitivity.** Each profile's sensitivity to a stressor is computed exactly as in the app (`src/lib/stressor-sensitivity.ts`): `sensitivity[s] = Σ_v polarity[v][s] · (score[v] − 3.5)/3.5`, using the ground-truth value→stressor polarity matrix in `src/lib/stressors.ts`. A positive value means the stressor *satisfies* the persona; negative means it *frustrates* them. Large differences **between** personas on a stressor are exactly the value tensions the app surfaces.

**Candidate pool.** 123 synthetic, psychologically-coherent value schemes were generated by sweeping "motivational directions" around the Schwartz circumplex (each profile peaks on one region of the value circle and troughs on its opposite), plus sharp single-value peaks and the four higher-order pure types. (A flat, all-3.5 profile was deliberately excluded: it has zero stressor sensitivity and so identifies nothing.)

**Selection.** Each profile is embedded as a combined feature vector — its 19 normalized value weights **and** its 12 stressor sensitivities, each block standardized and weighted equally. Portfolios are then chosen to maximize the **minimum pairwise distance** (max-min dispersion) in that combined space: the optimal core trio is found by exhaustive search, then extended by farthest-point selection. The result is **nested** — the 3-persona portfolio is the core of the 5, which is the core of the 12 — so you can scale up or down coherently.

**Spread & coverage achieved:**

| Portfolio | Min. pairwise distance | Stressors surfaced as tensions |
|---|---|---|
| 3 personas | 2.80 | **12 / 12** |
| 5 personas | 1.58 | **12 / 12** |
| 12 personas | 0.78 | **12 / 12** |

All three portfolios surface meaningful tension across **all 12 stressors** — even the compact trio. Larger portfolios add finer-grained value distinctions rather than new stressor coverage.

## Notes & tuning

- **Equal weighting** of value-space and stressor-space diversity (α = β = 1) was used. Re-running the optimizer with a heavier stressor weight would bias selection toward personas that react to *different* decision contexts even if their raw value profiles are closer; a heavier value weight does the opposite.
- **Synthetic, not real archetypes.** These profiles were generated to span the space, not matched to the 81 named archetypes in `src/lib/archetypes.ts`. Several do resemble familiar archetype families (e.g. *The Devoted* ≈ caregiver figures, *The Sovereign* ≈ power figures), and could be snapped to the nearest named archetype if recognizable names are wanted.
- **Reproducible.** The selection is deterministic given the polarity matrix; regenerating from the same data yields the same portfolios.
